## 1. Purpose

Quality here is **fitness for use** (Juran) in a money-meaning context: the output is a list of
probabilities and betting recommendations a human acts on. "Working" is not "tests pass" — it is
*"a number labelled value reflects a real edge, an uncertainty band means parameter uncertainty,
and the scorecard is computed against the right match."* Correctness must be built in (Deming):
because every defect in the probability→uncertainty→value chain is silent, inspection after the
fact cannot catch it — the output always looks plausible.

## 3. Coverage Theater Prevention

Project-specific fake-test patterns to reject:
- Asserting `sum(probs.values()) ≈ 1` **only** — normalization is trivially enforced by the code;
  a test that checks nothing else is theater. Pair every normalization check with an *ordering*
  or *value* assertion (the repo already does this — keep it).
- Testing `simulate` only at the default `runs=20000`. The interesting behavior (REQ-001 crash,
  REQ-002 band invariance) lives at the boundaries; a single mid-range run hides both.
- Building whale/holder fixtures with equal units so the CB-6 unit mix never surfaces.
- Mocking `_devig`/`_devig_shin` return values instead of feeding real odds — the bias-correction
  is the point.

## 4. Fitness-to-Purpose Scenarios

- **S1 — Overround removal.** Because raw odds embed a bookmaker margin, `_devig` must return a
  proper distribution preserving favorite order. *Verify:* `test_fitness_devig_removes_overround`.
- **S2 — Favorite-longshot correction.** Because longshots are systematically overpriced, Shin
  must lower the outsider and lift the favorite vs proportional. *Verify:* `test_fitness_shin_corrects_favorite_longshot`.
- **S3 — Draw inflation control.** Because independent Poisson underestimates draws, DC (ρ<0) must
  raise the draw a few points — not derail it. *Verify:* `test_fitness_dixon_coles_raises_draws`.
- **S4 — Whale containment.** Because a single extreme, unvalidated whale signal could fabricate
  edges, the gate caps its shift at 0.08. *Verify:* `test_fitness_whale_gate_caps_influence`.
- **S5 — Correlation cap.** Because multiple bets in one match are correlated, per-match stake is
  capped so one result can't sink the bankroll. *Verify:* `test_fitness_portfolio_correlation_cap`.
- **S6 — Stake ceiling.** Because model error is real, even a huge edge stays under the high-tier
  stake cap (quarter-Kelly + cap). *Verify:* `test_fitness_stake_capped_by_tier`.
- **S7 — Liquidity gate.** Because thin markets are untrustworthy, fewer than `MIN_BOOKS` books
  yields no value flag and zero stake. *Verify:* `test_fitness_liquidity_gate_blocks_thin_markets`.
- **S8 — MC ↔ analytic agreement.** Because the importance-weighted MC must equal the analytic DC
  distribution, the weighted draw probability matches within tolerance. *Verify:* `test_fitness_mc_matches_analytic_dixon_coles`.
- **S9 — Silent match loss (open risk).** Because the pipeline swallows per-match exceptions,
  a `simulate(runs<200)` crash drops the match with only a console line. *Verify (Phase 3):*
  regression test for CB-1.
- **S10 — Band-driven value drift (open risk).** Because the band feeds σ, raising `MC_RUNS`
  shrinks bands and loosens value flags. *Verify (Phase 3):* property test for CB-2.

## 5. AI Session Quality Discipline

- Never weaken a stake cap, liquidity gate, or correlation cap to make a test pass.
- When touching `monte_carlo` or `value_betting`, re-run `quality/test_functional.py` AND
  `tests/test_model.py` — the uncertainty plumbing couples them.
- Treat docstring contracts as authoritative: if the code diverges from the docstring (REQ-002,
  REQ-003), fix the code or change the docstring deliberately — do not let them drift.

## 6. The Human Gate

- Any change to staking math, the whale weight, or the data-quality tier thresholds is a human
  decision (real-money semantics). The disclaimer in `daily_matchday_run.py:225-226` must remain.
- Calibrating `ELO_PER_GOAL`, `DIXON_COLES_RHO`, or `ENSEMBLE_WEIGHTS` requires human review of
  `MODEL_VERSION` impact (cross-version comparability).
