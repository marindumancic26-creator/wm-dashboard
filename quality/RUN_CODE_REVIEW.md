# Code Review Protocol (three passes)

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.6 — Andrew Stellman
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit

Run each pass in a fresh context. Shared input: `quality/REQUIREMENTS.md`. Guardrails: line
numbers mandatory; read function bodies; grep before claiming "missing"; flag uncertainty as
QUESTION not BUG; no style-only findings.

## Pass 1 — Structural review
Mandatory scrutiny areas:
1. **Input validation / boundaries** — `monte_carlo.simulate` `runs` boundary (the
   `reshape(nb,chunk)` at `monte_carlo.py:97-99` for `runs<200`); `extract_1x2` outcome ordering
   (`polymarket_client.py:101`); `parse_slug` length/slice (`polymarket_client.py:59-66`).
2. **Resource lifecycle** — HTTP sessions/timeouts in clients; file writes in the pipeline
   (`daily_matchday_run.py:240-248`) — partial-write/overwrite of snapshots.
3. **Concurrency / state** — none material (single-threaded batch); state machine = data-quality
   tier; confirm all three tiers handled at every consumer.
4. **Unit / encoding correctness** — **whale exposure unit mix** (`whale_scoring.py:97-113`,
   share-count vs USD); λ units across `blend_lambdas`.
5. **Enumeration / whitelist completeness** — outcome key set `team1_win/draw/team2_win` across
   `features/ensemble/monte_carlo/value_betting/calibration`; verify `whale_signal`'s
   `if k in weighted` guard cannot drop a real outcome (already verified: keys align).

## Pass 2 — Requirement verification
For each REQ-001..012 cite ≥1 code location and a SATISFIED/VIOLATED verdict. Expected outcomes
from Phase 1 (verify, do not assume):
- REQ-001 VIOLATED at `monte_carlo.py:97-99` (runs<200). REQ-002 VIOLATED at `monte_carlo.py:96-105`.
- REQ-003 VIOLATED at `ensemble.py:74-89`. REQ-004 VIOLATED (integer line) at `value_betting.py:143-145`.
- REQ-005 AT-RISK at `value_betting.py:38,148-150`. REQ-006 VIOLATED at `whale_scoring.py:97-113`.
- REQ-008 VIOLATED at `data_quality.py:27`. REQ-009 VIOLATED at `ensemble.py:33`.
- REQ-010/012 SATISFIED (cite the gate/leakage code).

## Pass 3 — Cross-requirement consistency
Compare pairs sharing a field/constant:
- REQ-002 ↔ REQ-005: the band (REQ-002) feeds σ (REQ-005); if the band is noise, the σ derived
  from it is mis-scaled — confirm the coupling at `value_betting.py:121,148-150`.
- REQ-003 ↔ REQ-001: `blend_lambdas` output feeds `simulate`; inconsistent λ + crash boundary
  compound.
- REQ-009 ↔ REQ-AG2: the `0.2` vs `0.15` default changes the whale weight that the cap (REQ-AG2)
  then bounds.

## Output
Write confirmed bugs to `quality/BUGS.md` (`### BUG-NNN`), one regression-test patch per bug at
`quality/patches/BUG-NNN-regression-test.patch`, fix patches where small, and a confirmation
table. A code-path trace IS sufficient to confirm — do not defer for "runtime evidence".
