### BUG-001 ‚Äî `simulate` crashes for `runs < 200`, pipeline silently drops the match
- **Severity:** MEDIUM ¬∑ **Source:** code review (Pass 1/2) ¬∑ **File:** `monte_carlo.py:97-99`
- **Spec basis:** REQ-001; `simulate` is documented to return distributions for a run count.
- **Divergence:** `nb=200`, `chunk=max(1, runs//nb)`. For `runs<200`, `chunk=1` and

- **Code behavior (evidence):** `simulate(1.3,1.3,runs=50)` ‚Üí
  `ValueError: cannot reshape array of size 50 into shape (200,1)` (`BUG-001.red.log`).

  raises; inside `daily_matchday_run.py:186-189` the broad `except` turns it into a **silently
  dropped match** ‚Äî only a console line, no entry in the snapshot.
- **Disposition:** code-fix. **Fix type:** code.
```diff

+++ b/src/model/monte_carlo.py
@@ -95,7 +95,7 @@ def simulate(...):
```

### BUG-002 ‚Äî Uncertainty band measures simulation noise, not parameter uncertainty
- **Severity:** MEDIUM ¬∑ **Source:** code review (Pass 2/3) ¬∑ **File:** `monte_carlo.py:96-105`
- **Spec basis:** REQ-002; docstring `monte_carlo.py:3-6` ("5%/95%-Quantile ‚Ä¶ √ºber Parameter-Resamples").
- **Divergence:** every run draws an independent Gamma Œª (`monte_carlo.py:30-31`); the band then
  takes within-block 1X2 frequencies over arbitrary contiguous blocks of runs. A block is a
  mixed-Œª sample, so the across-block dispersion is dominated by MC sampling error and shrinks
  ~1/‚àöchunk as `MC_RUNS` grows.
- **Code behavior (evidence):** band width (team1_win, Œª=1.6/1.0, œÅ=0) = **0.1705** at runs=20000
  vs **0.0491** at runs=200000 (`BUG-002.red.log`). True parameter uncertainty (CV=0.12) would
  not shrink with more runs.
- **Observable consequence:** the band feeds œÉ ‚Üí `conservative_prob` ‚Üí `is_value`
  (`value_betting.py:121`). Raising `MC_RUNS` narrows bands ‚Üí shrinks œÉ ‚Üí loosens value flags.
  A performance knob silently changes betting recommendations.
- **Disposition:** code-fix. **Fix type:** code (redesign). Compute the band by resampling Œª at a
  *fixed* value per block and evaluating the 1X2 analytically/with shared draws, e.g.:
```diff
@@ def simulate(...):
-    bands = {}
-    nb = min(200, runs)
-    chunk = max(1, runs // nb)
-    idx = np.arange(runs)[: nb * chunk].reshape(nb, chunk)
-    wb = w[idx]
-    den = wb.sum(axis=1)
-    for name, cond in (("team1_win", g1 > g2), ("draw", g1 == g2), ("team2_win", g1 < g2)):
-        per = (wb * cond[idx]).sum(axis=1) / den
-        bands[name] = {"p05": ..., "p95": ...}
+    # Parameter-Unsicherheit: pro Resample EIN festes Lambda ziehen, 1X2 analytisch auswerten.
+    nb = 400
+    l1b = rng.gamma(k, lam1 / k, nb)
+    l2b = rng.gamma(k, lam2 / k, nb)
+    from src.model import features
+    per = {"team1_win": [], "draw": [], "team2_win": []}
+    for a, b in zip(l1b, l2b):
+        p1b, pdb, p2b = features.poisson_1x2(a, b, max_goals=mg, rho=rho)
+        per["team1_win"].append(p1b); per["draw"].append(pdb); per["team2_win"].append(p2b)
+    bands = {name: {"p05": round(float(np.quantile(v, 0.05)), 4),
+                    "p95": round(float(np.quantile(v, 0.95)), 4)} for name, v in per.items()}
```
- **Regression test:** `test_bug002_uncertainty_band_invariant_to_runs`.

### BUG-004 ‚Äî Integer Over/Under line assigns the push probability to "Under"
- **Severity:** MEDIUM ¬∑ **Source:** code review (Pass 1) ¬∑ **File:** `value_betting.py:143-145`
- **Spec basis:** REQ-004; `p_over + p_under + p_push = 1`.
- **Divergence:** `p_under = 1 - p_over` with `p_over = 1 - sum(total_dist[:int(ln)+1])`. For an
  integer line the exact-total push mass `P(total==ln)` is folded entirely into Under.
- **Code behavior (evidence):** line 3.0, Œª=1.4/1.2: push `P(total=3)=0.214`; code `p_under=0.734`
  vs correct `P(total‚â§2)=0.520` ‚Äî a **0.214 overstatement** of Under (`BUG-004.red.log`).
- **Observable consequence:** Under EV is biased up and Over down on whole-number totals. The Odds
  API usually returns `.5` lines (unaffected), so impact is conditional ‚Äî hence MEDIUM.
- **Disposition:** code-fix. **Fix type:** code (needs `import math`):
```diff
--- a/src/model/value_betting.py
+++ b/src/model/value_betting.py
@@
-        floor = int(ln)  # z.B. 2.5 -> over = total >= 3 = 1 - P(0..2)
-        p_over = round(1.0 - sum(total_dist[: floor + 1]), 4)
-        p_under = round(1.0 - p_over, 4)
+        # .5-Linien: kein Push. Ganzzahlige Linien: P(total==ln) ist Push -> ausschliessen.
+        p_over = round(1.0 - sum(total_dist[: int(ln) + 1]), 4)        # P(total > ln)
```
