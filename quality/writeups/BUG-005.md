# BUG-005 — Totals erhalten systematisch kleineres sigma als 1X2

> Generated by Quality Playbook v1.5.6 — https://github.com/andrewstellman/quality-playbook
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit · Severity: LOW · REQ-005

## 1. Summary
Totals erhalten systematisch kleineres sigma als 1X2.

## 2. Spec reference
REQ-005 — sigma muss markt-uebergreifend vergleichbar sein.

## 3. The code
Totals uebergeben band_width=None -> estimate_sigma Default bw=0.05 (value_betting.py:38,148-150).

## 4. Observable consequence
Gleicher Edge: 1X2 sigma=0.0655, Totals sigma=0.0616 -> Totals erreichen is_value leichter. Milde Bevorzugung -> LOW.

## 6. The fix
```diff
--- a/src/model/value_betting.py
+++ b/src/model/value_betting.py
@@ Totals-Band aus MC-Gesamttorverteilung statt Fixwert
-            r = _row('Ueber/Unter', nm, info['best_odds'], info['book'], p_model,
-                     info['consensus_prob'], None, info.get('n_books', 0))
+            tot_bw = 2 * (p_over * (1 - p_over)) ** 0.5 * 1.645   # ~90%-Band aus Bernoulli-sigma
+            r = _row('Ueber/Unter', nm, info['best_odds'], info['book'], p_model,
+                     info['consensus_prob'], tot_bw, info.get('n_books', 0))
```
