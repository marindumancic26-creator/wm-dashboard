# BUG-006 — Whale-Exposure mischt Anteilszahl (Holder) und USD (Trades)

> Generated by Quality Playbook v1.5.6 — https://github.com/andrewstellman/quality-playbook
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit · Severity: MEDIUM · REQ-006

## 1. Summary
Whale-Exposure mischt Anteilszahl (Holder) und USD (Trades).

## 2. Spec reference
REQ-006 — Exposure muss kommensurable (USD) Groessen kombinieren.

## 3. The code
Holder-Schleife addiert amount (Yes-Anteile, whale_scoring.py:97-104); Trade-Schleife addiert size*price (USD, :106-113) ins selbe Dict.

## 4. Observable consequence
800 Anteile @ 0.50 ($400) als 800 gezaehlt — gleich wie $800-Trade. Verzerrt Ranking/Gewichte/Specialist-Flag. Durch 0.08-Cap gedeckelt -> MEDIUM.

## 6. The fix
```diff
--- a/src/model/whale_scoring.py
+++ b/src/model/whale_scoring.py
@@ def whale_signal(snapshot, ...):
+    raw_price = snapshot['market'].get('raw', {})
     for outcome_key, hl in holders.items():
+        px = float(raw_price.get(outcome_key) or 0) or 1.0
         for h in hl[:config.WHALE_TOP_HOLDERS]:
             w = h.get('proxyWallet'); amt = float(h.get('amount') or 0)
             if w and amt > 0:
-                exposure[w][outcome_key] += amt
+                exposure[w][outcome_key] += amt * px   # USD
```
