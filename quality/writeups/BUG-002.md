# BUG-002 — Unsicherheitsband misst MC-Rauschen statt Parameter-Unsicherheit

> Generated by Quality Playbook v1.5.6 — https://github.com/andrewstellman/quality-playbook
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit · Severity: MEDIUM · REQ-002

## 1. Summary
Unsicherheitsband misst MC-Rauschen statt Parameter-Unsicherheit.

## 2. Spec reference
monte_carlo.py:3-6 / REQ-002 — Band = 5/95%-Quantil ueber Parameter-Resamples.

## 3. The code
Jeder Run zieht eigenes Gamma-Lambda (monte_carlo.py:30-31); Band nimmt Block-Frequenzen ueber Run-Bloecke (:96-105). Block = Misch-Lambda -> Dispersion ~ 1/sqrt(chunk).

## 4. Observable consequence
Bandbreite 0.1705 (runs=20000) -> 0.0491 (runs=200000). Band speist sigma -> conservative_prob -> is_value: MC_RUNS verschiebt still die Value-Empfehlungen.

## 6. The fix
```diff
--- a/src/model/monte_carlo.py
+++ b/src/model/monte_carlo.py
@@ Band aus festen Lambda-Resamples statt Run-Bloecken
-    nb = 200; chunk = max(1, runs // nb)
-    idx = np.arange(runs)[: nb*chunk].reshape(nb, chunk)
-    # ... Block-Frequenzen ...
+    nb = 400
+    l1b = rng.gamma(k, lam1/k, nb); l2b = rng.gamma(k, lam2/k, nb)
+    from src.model import features
+    cols = {o: [] for o in ('team1_win','draw','team2_win')}
+    for a, b in zip(l1b, l2b):
+        p1b, pdb, p2b = features.poisson_1x2(a, b, max_goals=mg, rho=rho)
+        cols['team1_win'].append(p1b); cols['draw'].append(pdb); cols['team2_win'].append(p2b)
+    bands = {o: {'p05': round(float(np.quantile(v,0.05)),4),
+                 'p95': round(float(np.quantile(v,0.95)),4)} for o,v in cols.items()}
```
