# BUG-003 — blend_lambdas verwirft Buecher-Lambda wenn Polymarket fehlt

> Generated by Quality Playbook v1.5.6 — https://github.com/andrewstellman/quality-playbook
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit · Severity: LOW-MED · REQ-003

## 1. Summary
blend_lambdas verwirft Buecher-Lambda wenn Polymarket fehlt.

## 2. Spec reference
REQ-003 — bei fehlendem market muss Lambda die gleichen bepreisten Quellen nutzen wie blend_probs.

## 3. The code
blend_lambdas baut Kandidaten nur aus market+model (ensemble.py:74-81); books/kalshi-Gewicht faellt an w_market ohne Kandidat.

## 4. Observable consequence
Books treiben Probs (team1_win 0.51), aber blend_lambdas liefert 1.30/1.30 (nur Modell) -> MC-Torverteilung inkonsistent. Discovery Polymarket-getrieben -> selten -> LOW-MED.

## 6. The fix
```diff
--- a/src/model/ensemble.py
+++ b/src/model/ensemble.py
@@ def blend_lambdas(market, model, weights_used):
     if market and market.get('probs'):
         mi = features.market_implied_lambdas(...); cands['market'] = (mi['lambda1'], mi['lambda2'], mi)
+    elif books and books.get('probs'):
+        mi = features.market_implied_lambdas(books['probs']['team1_win'], books['probs']['draw'], books['probs']['team2_win'])
+        cands['market'] = (mi['lambda1'], mi['lambda2'], mi)
```
