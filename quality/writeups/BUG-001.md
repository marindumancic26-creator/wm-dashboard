# BUG-001 — simulate() crasht bei runs<200 -> stiller Match-Verlust

> Generated by Quality Playbook v1.5.6 — https://github.com/andrewstellman/quality-playbook
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit · Severity: MEDIUM · REQ-001

## 1. Summary
simulate() crasht bei runs<200 -> stiller Match-Verlust.

## 2. Spec reference
monte_carlo.py:3-6 / REQ-001 — simulate liefert Verteilungen fuer einen Run-Count.

## 3. The code
nb=200, chunk=max(1,runs//nb). Fuer runs<200 ist nb*chunk=200>runs, also arange(runs)[:200].reshape(200,1) -> ValueError. (monte_carlo.py:97-99)

## 4. Observable consequence
Jeder Aufruf mit runs<200 (Draft, Backtest-Sweep) wirft; im Pipeline-except (daily_matchday_run.py:186-189) wird das Spiel STILL verworfen.

## 6. The fix
```diff
--- a/src/model/monte_carlo.py
+++ b/src/model/monte_carlo.py
@@ def simulate(...):
-    nb = 200
+    nb = min(200, runs)
     chunk = max(1, runs // nb)
```
