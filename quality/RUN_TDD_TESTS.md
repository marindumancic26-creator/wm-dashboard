# TDD Verification Protocol

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.6 — Andrew Stellman
> Date: 2026-06-14 · Project: wm-dashboard-fussball-wahrscheinlichkeit

For each confirmed bug in `quality/BUGS.md`, prove it with a red→green cycle. The math is
deterministic given a seed, so every candidate here is a pure unit test (no HTTP, no fixtures
beyond constructed dicts).

## Framework
Same as the project: `assert`-style tests runnable under pytest. Place regression tests in
`quality/test_regression.py` with `@pytest.mark.xfail(strict=True, reason="BUG-NNN: ...")`.
Import pattern: `sys.path.insert(0, repo_root)` then `from src... import ...`.

## Per-bug procedure
1. Read the bug's REQ + spec basis (docstring/line).
2. Write a test whose assertion echoes the *intended* contract (spec language, not code naming).
3. **Red:** run against current code — must FAIL. Log to `quality/results/BUG-NNN.red.log`
   (first line `RED`).
4. **Green:** apply `quality/patches/BUG-NNN-fix.patch`, re-run — must PASS. Log to
   `quality/results/BUG-NNN.green.log` (first line `GREEN`). Revert the fix; keep the xfail guard.

## Candidate-specific test sketches
- **BUG (CB-1) / REQ-001:** `simulate(1.3,1.3,runs=50)` must return a normalized result (currently
  raises `ValueError` on reshape). Red asserts "no exception + sum≈1".
- **BUG (CB-2) / REQ-002:** band width at `runs=20000` ≈ width at `runs=200000` for fixed λ
  (within tolerance). Red shows the high-runs band materially narrower.
- **BUG (CB-3) / REQ-003:** with `market=None, books=<probs>, model=<m>`, `blend_lambdas` λ must
  move toward the book-implied λ. Red shows it equals the model-only λ.
- **BUG (CB-4) / REQ-004:** for a `3.0` totals line, `p_under` must equal `P(total≤2)`. Red shows
  it equals `1 - P(total≥4) = P(total≤3)` (push mass leaked).
- **BUG (CB-5) / REQ-005:** at equal `(p_model,p_market,n_books)`, a totals `_row` must not reach
  `is_value` at a lower true confidence than a 1X2 `_row`. Red shows the asymmetry from `bw=0.05`.
- **BUG (CB-6) / REQ-006:** a holder-heavy vs trade-heavy snapshot with equal *USD* must produce
  equal exposure. Red shows share-count holders dominating.

## Run command
`python -m pytest quality/test_regression.py -q` (xfail expected = XFAIL; after a fix the marker's
strict mode turns an accidental pass into a failure, flagging the guard for removal).
